# 简易版_计算机发展历史

`计算机`也就是电脑，最开始是被制造用来战争中计算`导弹轨迹`、`破解加密数据`的，后面战争结束了，国家已经投入了几百亿 又不能白白浪费，就投入到`民用`，发展经济。

## 第一阶段

一开始的电脑很庞大，需要在机器上转动齿轮，一个齿轮代表一个英文字母。

比如输入导弹的地址、速度，就能自动计算出要掉落在哪里。

缺点：
- 占地很大
- 操作麻烦，有几百个齿轮要转动，很容易出错

## 第二阶段

为了优化第一阶段的操作麻烦问题，也为了`保存程序设计，重复使用`，出现了`穿孔纸带`

![穿孔纸带](https://ss2.baidu.com/-vo3dSag_xI4khGko9WTAnF6hhy/baike/pic/item/a2cc7cd98d1001e95c613526ba0e7bec54e797bd.jpg)

它的原理是`二进制(这又是一个新名词)`，简单来说  二进制的所有数据都是用0、1  两个数据来表示（也就是纸带上的 有打孔 没打孔 两个状态）

假设我们约定好了 以下规则（举例的，不是真实的）
- 00000001  代表字母A
- 00000010  代表字母B
- 00000100  代表字母C
- 00001000  代表字母D
- 00010000  代表字母E


那么就能把数据 打在纸条上了 

优点：
- 纸带不丢掉，不出错的情况下，下次要执行同一个程序，就可以继续使用
- 纸带轻便，成本少

缺点：
- 打孔打错了，要更改很麻烦，比如打错了一个孔，整个程序都要重头打一次纸条（很长很长的程序就要浪费很多纸）


## 第三阶段

为了解决第二阶段的问题，开始设计了`把数据存到磁盘上（电脑的硬件）`，并且是可以只修改一小部分

我们电脑、手机上用到的所有软件、文件，都是保存在磁盘上的（手机常说的 `存储空间`、`内存`）

比如我们设计了一个截图的导弹程序，保存为txt文件

![导弹防卫系统](https://s4.ax1x.com/2022/01/21/7gqmct.png)
![导弹防卫系统](https://s4.ax1x.com/2022/01/21/7gqe1I.png)


但是电脑的设计，电路板只能识别出`有通电`、`没通电`的情况，所以还是用`2进制`的存储的

一个小零件的状态代表一个纸条上的小孔 跟第二阶段的原理是一样的

![导弹防卫系统](https://gimg2.baidu.com/image_search/src=http%3A%2F%2F05imgmini.eastday.com%2Fmobile%2F20200610%2F20200610200326_3039ad811468cda2c4587b911bb706d3_2.jpeg&refer=http%3A%2F%2F05imgmini.eastday.com&app=2002&size=f9999,10000&q=a80&n=0&g=0n&fmt=jpeg?sec=1645322145&t=2cf83e32b22fa63525389bdf7e534521)

优点：
- 解决了程序的存储、局部修改，有利于复杂程序的开发

缺点：
- 需要先解决`编程语言`的设计，转化为`2进制`，解决磁盘的储存设计等等（这得益于行业的发展，所以才能利用这些零件，组装成第三阶段的计算机）

## 第四阶段

还是用上面的例子，我们的电脑  发展到后面，已经不仅仅是一个人办公了，

假设我们要去支援朝鲜，那除了军队，我们还要通知其他同事，做好战争的准备（比如 粮食部门，做好屯粮）

所以要解决`文件共享`的问题，军队在txt里写下 他们执行任务的日志

```
2022-1-21 10:13分，检测到美国导弹信号，进入战争状态
```

粮食部门的人 每天都会来查看这个文件，确定是否要进入战争状态。

传统的做法是把整个`日志.txt`都发给每个部门，但是这样子很容易出错，因为每天都有一个新的文件产生，很容易弄乱。

最好的做法是，每个部门都来查看这个txt。（当然不是走路过来看屏幕哦，而是在别的电脑上  直接查看这个文件）

那我们就可以通过`网络技术`共享信息了。久而久之就形成了`互联网`（每一台计算机，只要插了网线，就加入了全球的网络，可以访问别人的资源[只要对方允许]）

所以进入到了后面的阶段，军队部门做了一个网站，每天都在网站上更新日志，其他部门（包括老百姓） 都可以打开网址来查看最新的消息，提高了信息共享的效率！

## 学习 内容

话题回到我们的学习上，我们要学的就是html，怎么样做网站，来展示信息！给客户 或者给公司内部的同事查看。


### html
html是一种约定传输协议，其实它的本质跟上面说的txt是一样的，只是加多了一些`标识`

这样子`浏览器软件`会对信息做出特定的处理  比如字体的大小
- 这也是可以更改的，这属于下面学的部分——css
- 浏览器默认内置自带了一部分的css。我们可以重新写一个新的覆盖它
