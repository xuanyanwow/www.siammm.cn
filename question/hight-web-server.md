# 高可用 ——  Web


## 客户端到web-server

今天在看文章  一个高可用架构的知识，说到web-server的高可用  可以使用keepalived来部署

这里会产生一个疑问，绑定的VIP(虚拟IP)，如果是公网IP，公网IP和网卡之间的绑定关系是如何处理

最后了解到硬件的知识，NAT  IP转化技术，可以购买NAT服务器或者服务

使公网IP  比如 88.56.561.12   转化为固定的内网IP   10.0.0.44

[阿里云NAT网关](https://www.aliyun.com/product/network/nat?spm=5176.19720258.J_3207526240.35.e93976f4M3qR7R)

然后就可以在keepalived 绑定虚拟内网IP   达到多台服务器 漂移服务的功能
