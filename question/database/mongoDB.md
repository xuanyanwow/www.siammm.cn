# mongoDB

##同样是nosql   跟redis 、memcache有啥区别？

1 类型
- redis memcache都是内存型缓存数据库
- mongoDB是基于分布式文件存储   文档型数据库（介于关系型和非关系型之间）


2 事务（数据一致性）
- redis 只能保证一个客户端连接的事务中的操作连续执行
- memcache用cas保证数据一致性
- mongoDB没有事务


3 应用场景

- redis  数据缓存  同时有数据类型较多、数据运算的需求
- memcache  数据缓存  没有运算需求  性能更高
- mongoDB 数据库 大量数据的检索


4 可靠性（持久化）

- 对于数据持久化和数据恢复，Redis 支持（`快照`、`AOF`）：依赖快照进行持久化，aof 增强了可靠性的同时，对性能有所影响。
- memcache 不支持，通常用在做缓存, 提升性能；
- MongoDB 从 1.8 版本开始采用 binlog 方式支持持久化的可靠性。

5 可用性（单点）

- Redis 对于单点问题，依赖客户端来实现分布式读写；主从复制时，每次从节点重新连接主节点都要依赖整个快照, 无增量复制，因性能和效率问题，所以单点问题比较复杂；不支持自动 sharding, 需要依赖程序设定一致 hash 机制。一种替代方案是，不用 Redis 本身的复制机制，采用自己做主动复制（多份存储），或者改成增量复制的方式（需要自己实现），一致性问题和性能的权衡。
- Memcache 本身没有数据冗余机制，也没必要；对于故障预防，采用依赖成熟的 hash 或者环状的算法，解决单点故障引起的抖动问题。
- mongoDB 支持 master-slave,replicaset（内部采用 paxos 选举算法，自动故障恢复）,auto sharding 机制，对客户端屏蔽了故障转移和切分机制。


## 既然对标的是数据库，跟mysql的对比如何


mongodb没有事务支持，数据结构也更简单，所以性能更好

mongodb如果没有对主键设置值，则插入的效率会最慢（内部自行计算）

mysql不管没有设置主键相对也更快，差别不大

mongodb占用的空间更大，支持全索引（文档内部字段的索引） 检索更灵活更快

没有关系型数据库join等关系
