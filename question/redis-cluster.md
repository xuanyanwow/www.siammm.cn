# redis集群

## 拆分槽存储


一个redis集群包含16834个哈希槽（hash slot）数据库中的每个键都属于这 16384 个哈希槽的其中一个， 集群使用公式 `CRC16(key) % 16384 `来计算键 key 属于哪个槽


如果用户将新节点 D 添加到集群中， 那么集群只需要将节点 A 、B 、 C 中的某些槽移动到节点 D 就可以了。

`扩展方便`


## 集群主从复制


如果只有集群，某个节点挂了，那么这部分槽的数据就丢失了

所以每一个节点都需要配置从节点，主从复制，自动切换为主，接管工作


## 数据一致性


redis 异步复制，所以是数据不强一致的分布式
