# 中介模式

- 尽管将一个系统分割成许多对象通常可以增加其`可复用性`，但是对象间相互连接的激增又会降低其`可复用性`了
- 大量的连接使得2一个对象不可能在没有其他对象的支持下工作，系统表现为一个不可分割的整体，所以对系统的行为进行任何较大的改变就十分困难了


网状结构->以中介者为中心的星型结构

> 用一个中介对象来封装一系列的对象交互。中介者使各对象不需要显式地相互引用，从而使其耦合松散，而且可以独立的改变它们之间的交互

## 思考

- 游戏Server中，中介者来处理各玩家出牌数据



## 优缺点

- 很容易在系统中使用，也很容易被误用。当系统出现了`多对多`交互复杂的对象群时，不要急于使用中介者模式，而要先反思你的系统在设计上是否合理
- 中介者的职责过多，中介者比任何一个成员都复杂
- 中介者模式一般应用于一组对象以`定义良好`但是`复杂的方式进行通信`的场合
- 想定制一个分布在多个类中的行为，而又不太想生成太多的子类的集合
