# 迪米特法则

> 如果两个类不比彼此直接通信，那么这两个类就不应当发生直接的相互作用。如果其中一个类需要调用另一个类的某一个方法的话，可以通过第三者转发这个调用。

- 在类的结构设计上，每一个类都应对尽量`降低`成员的访问权限
- 强调了类之间的松耦合
- 类之间的耦合越弱，越有利于复用，一个处在弱耦合的类被修改，不会对有关系的类造成波及。（隐藏信息促进了复用）

新员工到IT部报道，人事给你直接指派的同事需要出差，所以你干等了一天，找其他同事也不愿意给你领取电脑（人事安排的不是我）

引出两个概念

- 如果公司给你指派的不是同事，而是第三者的主管，那么主管在中间可以安排所有员工给你领取办理
- 如果主管也不在，那么用什么策略，在公司里谁管事（负载均衡的选主逻辑）
