# 观察者模式

> 定义了一种一对多的依赖关系，让多个观察者对象同时监听某一个主题对象。这个主题对象状态发生变化时，会通知所有的观察者对象，使它们能够自动更新自己

PHP框架中常见的Event、Hook 就是一种观察者模式

```php
Event::("create_order", function(){
   echo 1; 
}); 
```
## 特点

- 将一个系统分割成一系列互相协作的类有一个很不错的副作用，那就是需要维护相关对象之间的一致性。我们不希望为了维持一致性而使得各类紧密耦合，这样子会给维护、扩展和重用带来不便。而发布-订阅模式  使得两者都不需要知道对方的存在，只关心数据。
- 当一个对象的改变需要同时改变其他对象的时候，而且它不知道具体有多少对象待改变，应当考虑观察者模式
- 一个抽象模式有两个方面，其中一方面依赖于另一方面，这时用观察者模式可以将这两者封装在独立的对象中使它们各自独立地改变和复用
- 观察者模式在解除耦合，让双方都依赖于抽象，而不是依赖于具体。


## 应用场景

- 订单创建时，通知数据统计观察者、仓库出货观察者、语音播报观察者...
- 司机接单时，通知乘客观察者、平台观察者、计费观察者、路线规划观察者

```php

// 伪代码

// 创建订单
public function create()
{
    // 校验参数
    // ..
    
    // 校验用户权限
    // .. 
    
    // 插入订单表 生成Model
    // .. 
    
    // 订单通知者（订单发布者）
    
    $order_subject = new OrderSubject($order_model);
    $order_subject->addObserver(new DataStasticsObserver());
    $order_subject->addObserver(new ErpObserver());
    $order_subject->addObserver(new MerchantVideoServerObserver());
    $order_subject->create();// 告诉所有观察者 订单创建了
    
}

```

## 升级

委托行为  不强求传递addObserver()的是一个Server类

php中可以用回调闭包来处理   比如
```php
Event::("create_order", function(){
   echo "统计数据到redis"; 
}); 

Event::("create_order", [ErpObserver::class, "create_order"]); 
```
