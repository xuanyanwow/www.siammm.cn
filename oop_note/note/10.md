# 建造者模式(生成器模式)

不同厨师、同一个厨师 炒出来的饭菜味道不稳定，违法了`依赖倒转原则`（抽象不应该依赖细节，细节应该依赖于抽象）

具体的厨师应该依赖于操作标准，操作标准不应该为某一个厨师做出变化。

肯德基炸鸡的时间都是机器固定的，不会因为某个地区和人做出变化

## 画游戏人物

人物都必须有头、身体、左手、右手、左脚、右脚。这里建造小人的过程是稳定的，具体的细节是不同的

如果需要`将一个复杂对象的构造与它的标识分离，使得同样的构造过程可以创建不同的表示`时，我们可以用`建造者模式(Builder)` 又叫生成器模式

## 建造者模式定义

> 将一个复杂对象的构造与它的表示分离，使得同样的构造过程可以创建不同的表示

## 代码实现

- 1.构建过程用一个抽象类固定住，每一个部件声明一个抽象方法
- 2.具体的产品实现抽象类
- 3.增加一个指挥者(Directore) 控制建造过程，也是用它来隔离用户与建造过程的关联

指挥者->建造者(依赖一个抽象建造者 可以定义建造者的生产流程)->得到产品
    

```php
// 炸鸡指挥者
class FriedChickenDirector{
    private $fried_chicken_product;
    
    public function __construct(FriedChicken $fried_chicken_product)
    {
        $this->fried_chicken_product = $fried_chicken_product;
    }
    
    public function create_fried_chicken()
    {
        $this->fried_chicken_product->bamao();// 鸡拔毛
        $this->fried_chicken_product->clear();// 清洗
        $this->fried_chicken_product->yanzhi();// 腌制
        $this->fried_chicken_product->zha();// 油炸
        $this->fried_chicken_product->fire();// 火侯...
    }
    
}
```

客户端调用
```php
// 黑椒炸鸡
$fried_chicken_heijiao = new FriedChickenHeijiaoBuilder();
$fried_chicken_heijiao_director = new FriedChickenDirector($fried_chicken_heijiao);
$fried_chicken_heijiao_director->create_fried_chicken():

// 麻辣炸鸡
$fried_chicken_mala = new FriedChickenMalaBuilder();
$fried_chicken_mala_director = new FriedChickenDirector($fried_chicken_mala);
$fried_chicken_mala_director->create_fried_chicken():
```
