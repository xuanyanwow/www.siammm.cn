# 命令模式

> 将一个请求封装为一个对象，从而使你可用不同的请求对客户进行参数化；对请求排队或记录请求日志，以及支持可撤销的操作

- 由于行为请求者和行为实现者，紧耦合，容易出现问题
- 需要对请求排队、记录请求日志、支持可撤销的操作时，紧耦合是不合适的


## 作用

- 很容易设计一个命令队列；
- 在需要的情况下，可以较容易地将命令计入日志
- 允许接受请求的一方决定是否要否决请求
- 可以容易地实现对请求的撤销和重做
- 由于加进新的具体请求类不影响其他类，隐藏增加新的具体命令类很容易
- 命令模式把请求一个操作的对象与知道怎么执行一个操作的对象分割开

## 思考

防火墙
- 请求进来先排队，记录日志。请求可以取消（取消后服务员得到响应后 直接丢弃）
